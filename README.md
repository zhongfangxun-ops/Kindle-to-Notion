# Kindle 笔记导入 Notion - 极致健壮版

## 功能特点

✅ **弹性多源读取**：自动识别并处理 TXT 和 HTML 格式的 Kindle 笔记  
✅ **智能去重**：基于内容包含关系的去重，保留最完整的笔记  
✅ **位置排序**：按原文位置自动排序，容错处理异常位置信息  
✅ **接续编号**：增量导入时自动从已有笔记数量+1开始编号  
✅ **安全写入**：只追加新内容，绝不修改已有页面内容和自定义属性  
✅ **分批处理**：自动处理 Notion API 100 blocks 限制  
✅ **全局容错**：完善的异常处理，确保程序不会因单个错误而崩溃

## 使用步骤

### 1. 安装依赖

```bash
pip install requests beautifulsoup4
```

### 2. 配置 Notion

在脚本中修改以下配置：

```python
NOTION_TOKEN = "your_notion_integration_token_here"
DATABASE_ID = "your_database_id_here"
```

**获取 Token 和 Database ID：**
- Token：访问 https://www.notion.so/my-integrations 创建 Integration
- Database ID：打开数据库页面，URL 中的 32 位字符串

**数据库属性要求：**
- `Names`（Title 类型）
- `Author`（Rich Text 类型）

### 3. 准备笔记文件

将以下文件放在脚本同目录下：
- `My Clippings.txt`（Kindle 导出的笔记文件）
- 或/和 `.html` 文件（app端Kindle 笔记导出的 HTML 格式）

### 4. 运行脚本

```bash
python kindle_to_notion.py
```

## 核心逻辑说明

### 文件检测
- 仅有 TXT → 处理 TXT
- 仅有 HTML → 处理 HTML  
- 都有 → 合并处理
- 都没有 → 友好提示并退出

### 容错排序
从位置信息中提取首个数字作为排序依据，提取失败时默认为 999999，确保不会因 NoneType 导致崩溃。

### 智能编号
- 新建页面：从 1 开始
- 已有页面：先统计现有 quote block 数量 N，从 N+1 开始

### 数据库保护
- 创建页面时只填写"Names"和"Author"，不干涉其他自定义属性
- 使用 Append API 追加到页面底部，不修改现有内容

### 美化排版
每条笔记转为 quote block，包含：
- 第一段：序号. 正文（黑色）
- 第二段：\n[位置信息]（灰色斜体）

## 注意事项

1. 首次运行前务必配置 NOTION_TOKEN 和 DATABASE_ID
2. 确保 Notion Integration 已添加到目标数据库
3. 数据库必须包含"书名"和"作者"属性
4. 脚本会自动处理 API 限流，无需手动干预
5. 支持多次增量导入，不会重复编号

## 故障排查

**问题：提示缺少依赖库**
```bash
pip install requests beautifulsoup4
```

**问题：找不到文件**
确保笔记文件与脚本在同一目录下

**问题：API 调用失败**
- 检查 Token 是否正确
- 确认 Integration 已添加到数据库
- 检查网络连接

**问题：属性名称不匹配**
修改脚本中的属性名称以匹配你的数据库：
```python
'properties': {
    '书名': {  # 改为你的标题属性名
        'title': [...]
    },
    '作者': {  # 改为你的作者属性名
        'rich_text': [...]
    }
}
```
